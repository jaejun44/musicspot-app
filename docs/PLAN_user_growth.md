## 우선순위 (ROI 순)

### 1. 바이럴 버그 수정 — 로그인 returnTo + count 증가 ★최우선
- **문제**: 8마디 공유 링크를 받은 비로그인 사용자가 "이어 만들기"→`/login`→로그인 후 **홈으로 튕김**(공유받은 프로젝트로 복귀 못 함). K-factor 루프가 여기서 끊김.
- **문제2**: `share_count`/`pass_count`가 코드 어디서도 증가 안 함 → "인기 챌린지" 정렬(`generateStaticParams share_count desc`)·OG가 항상 0 기준으로 죽어 있음.
- **수정**: 
  - `StemsClient.openCreate()` / 업로드 게이트에서 `/login?returnTo=/stems/{id}` 로 넘김
  - `LoginClient` 가 `returnTo`/`next` 파라미터를 받아 `/auth/callback` 후 복귀
  - `handleShare` 에서 `share_count` increment RPC, 트랙 업로드 성공 시 `pass_count` increment
  - 비로그인 모달의 수동 안내문을 "이어서 8마디 만들기 →" 버튼으로 교체
- 작업량: 1~1.5일

### 2. /region 허브 인덱스 페이지 — SEO 고아 구제 ★가성비 최고
- **문제**: `/region` 인덱스 없음 → 인천·수원·대전 지역 페이지가 고아(내부링크 거의 없음). 도시 단위 키워드("서울 연습실") 랜딩 없음.
- **수정**: `app/region/page.tsx` 신설. REGIONS를 도시별 그룹핑 + 전체 25개 링크 + 도시 h2. sitemap·푸터·breadcrumb에 `/region` 연결.
- 작업량: 반나절

### 3. 신규 방문자 온보딩 — 딥링크 진입자 가치 전달
- **문제**: SEO·검색 딥링크 진입자는 "여기 뭐 하는 곳" 설명을 못 받음. "8마디가 뭔지" 설명 없음. (기존 OnboardingModal은 이름과 달리 로그인 후 밴드매칭 프로필 폼일 뿐, 신규 방문자 온보딩 아님)
- **수정**: 검색/지역 페이지에 1줄 가치 스트립 또는 1회성 코치마크. `/stems` 첫 진입 시 "8마디란?" 30초 데모/예시.
- 작업량: 1일

### 4. 지역 확장 — 누락 밴드씬 8~12개 + 고유 콘텐츠
- **문제**: 25개가 서울 편중(20). 광주·울산 광역시 누락. 25개가 거의 중복 템플릿 한 줄.
- **수정**: region-alias·REGIONS에 광주·울산·전주·천안·부천·안양·영등포·신도림 등 추가. regions.ts에 `intro`·`faqs` 필드 + FAQPage JSON-LD.
- 작업량: 1~2일

### 5. 8마디 명예/진행 가시화 — 재공유 욕구
- **문제**: 점수·명예·진행 가시화 부재. 자랑할 결과물 없음.
- **수정**: "N번째 주자", 참여자 N명/진행바, 참여자 이모지 스택. 완성 시 축하+공유 산출물.
- 작업량: 2~3일
